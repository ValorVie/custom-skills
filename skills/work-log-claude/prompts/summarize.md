# 工作日誌摘要指引

根據提供的 JSON 資料中的 `projects` 和 `summary` 欄位，產生工作日誌摘要。

## 當日總結（必要）

用一句話（50-100 字）概括全部工作重點。
- 聚焦在「做了什麼」和「為了什麼目的」
- 提及涉及的主要專案名稱
- 不要列舉，要有敘事感

## 逐專案摘要（必要）

對每個有活動的專案，產生 2-5 個要點：
- 分析 `git_commits` 的 message 和 diff stat（files_changed / insertions / deletions），理解變更的語意
- 將相關的 commits 歸類為一個要點（不要逐 commit 列出）
- 每個要點用一句話描述做了什麼、改了什麼
- 如果該專案沒有 `git_commits`，根據 session 數量和 `duration_minutes` 推斷為研究/探索活動
- 如果資訊不足以產生有意義的摘要，標記為「研究/探索 session（N 分鐘）」
- 使用繁體中文

## 輸出格式

嚴格按以下格式輸出，不要加其他標題或說明：

```
### 當日總結

<一句話摘要>

### 逐專案摘要

#### <project short_name>
- <要點 1>
- <要點 2>
- ...
```
