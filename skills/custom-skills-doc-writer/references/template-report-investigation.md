# 模板：調查報告

```markdown
---
title: {title}
type: report/investigation
date: {date}
author: {author}
status: draft
---

# {title}

## 摘要

一段話描述調查結論。

## 調查背景

### 觸發原因
- 事件/問題描述
- 發現時間與方式

### 調查範圍
- 涉及的系統/模組
- 調查時間範圍

## 調查方法

1. 方法 1（日誌分析/程式碼審查/重現測試/...）
2. 方法 2

## 發現

### 發現 1: {標題}

**嚴重度**: 高/中/低
**證據**:
- 程式碼片段、日誌、截圖等

**分析**:
詳細說明發現的內容及其影響。

### 發現 2: {標題}

（同上格式）

## 根因分析

### 直接原因
- 描述直接導致問題的因素

### 根本原因
- 描述背後的系統性問題

### 因果鏈

```
根本原因 → 中間因素 → 直接原因 → 症狀/問題
```

## 影響評估

| 面向 | 影響 | 說明 |
|------|------|------|
| 使用者 | 高/中/低/無 | ... |
| 資料 | 高/中/低/無 | ... |
| 安全 | 高/中/低/無 | ... |
| 效能 | 高/中/低/無 | ... |

## 建議

| 優先級 | 建議 | 理由 | 預估工作量 |
|--------|------|------|------------|
| P0 | ... | ... | ... |
| P1 | ... | ... | ... |
| P2 | ... | ... | ... |

## 結論

總結調查結果與下一步行動。

## 附錄

- 原始日誌
- 相關程式碼連結
- 參考文件
```
